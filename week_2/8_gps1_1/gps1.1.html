Pair Programming

Kenneth Mendonca and Timoor Kurdi

What is it?

  • Collaboratively working on one project on the same computer
  • There is a driver who writes the code
  • The navigator reviews each line of code and directs the workflow
  • Though there are two different people, they mutually work towards the same goal

Why is it important?

  • Enhances learning ability and speeds up the debugging process
  • Mimicks real world working scenarios
  • Produces cleaner and more efficent code
  • Helps team members share problems and come up with solutions more quickly

Our take away from pair programming

Pairing for the first time was fun and difficult at the same time. We enjoyed because it was a lot easier when something wasn't working to figure it out what did work. Giving each other time and using each other's input to create good working code was definitely a plus, but working remotely was a challenge. Overall, it definitely was a good experience to pair program as we learned a lot together than we would have by ourselves.
